# TASKS: AI-ментор, связи тем и проверка по эталону

Источник: `docs/prd-ai-mentor-reference-check.md`

Цель работ: добавить в Analyst Trainer слой учебных связей и AI-ментора, который проверяет ответы по эталону, объясняет ошибки, дает подсказки без раскрытия решения и предлагает следующий шаг. AI не должен заменять точную автопроверку, хранить секреты во frontend-коде или менять прогресс без явного действия пользователя.

## Риски и контрольные меры

- [x] AI может дать уверенный, но неверный разбор: всегда показывать связь с эталоном, rubric и статус "feedback не является автозачетом".
- [x] Пользователь может копировать ответы AI: режим `hint` не получает полный `reference`, полный эталон показывать только в явном режиме.
- [x] Proxy может потратить лимит: backend rate limit обязателен в MVP.
- [x] Контекст может быть слишком большим: отправлять только нужные поля под конкретный режим.
- [x] Topic graph может устареть: добавить data validation и предупреждения о битых refs.
- [x] AI может случайно изменить прогресс: progress update только через явное действие пользователя.
- [x] Офлайн-режим может сломаться: no-proxy сценарий обязателен в smoke-проверке.
