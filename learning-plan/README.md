# Learning Plan

Эта папка хранит исходные учебные документы и структурированные данные для интеграции учебной системы в Analyst Trainer.

`data/` является единственным источником учебного контента для UI. Пользовательский прогресс, статусы, заметки, отклики и настройки обучения хранятся отдельно в IndexedDB.

## Структура

```text
learning-plan/
  source/          исходные .docx файлы
  data/            JSON-данные, которые будет читать приложение
  validate-data.mjs проверка контентного контракта MVP
```

## Источники

- `source/План_Data_Analyst_7_месяцев_UPDATED.docx` — основной 7-месячный план.
- `source/enhancement_pack.docx` — пакет усиления: дневная каденция, чек-листы, mock-интервью, карьерные блоки.

## Данные для приложения

- `data/plan.json` — источник для разделов "Сегодня" и "План": 7 месяцев, 14 спринтов, недели, темы, проекты, карьерные действия и связи с модулями тренажера.
- `data/tasks.json` — источник для раздела "Задачи": 100 обязательных практических задач, дополнительные задачи, навыки, месяцы, допустимые статусы и связи с модулями.
- `data/projects.json` — источник для раздела "Проекты": 3 якорных проекта портфолио, чек-листы качества, deliverables и дефолтная структура UI-полей.
- `data/career.json` — источник для раздела "Карьера": карьерные действия, старт активных откликов с месяца 4, структура трекера вакансий, ATS-резюме и mock-интервью.

Приложение работает с JSON и не читает `.docx` во время работы в браузере. `.docx` остаются только первоисточниками для ручного обновления JSON.

## Разделение контента и прогресса

JSON хранит учебный контент и дефолтные схемы:

- месяцы, недели, темы и проекты;
- названия задач и связи с модулями;
- списки допустимых статусов;
- дефолтные поля формы проекта в `statusFields`;
- структуру полей карьерного трекера.

IndexedDB хранит реальные пользовательские данные:

- дату старта и режим расписания;
- выполненные элементы дня;
- дневниковые записи;
- статусы задач;
- прогресс проектов, ссылки, заметки и чек-листы;
- отклики на вакансии и их статусы.

Важно: `statusFields` в `projects.json` — это шаблон формы, а не прогресс пользователя. Реальные значения должны записываться только в IndexedDB.

## MVP-покрытие

Текущий контент покрывает MVP-разделы:

- "Сегодня" — строится из `plan.json`, даты старта и пользовательского состояния.
- "План" — берет месяцы, спринты, недели, темы, проекты и карьерные действия из `plan.json`.
- "Задачи" — берет обязательные и дополнительные задачи из `tasks.json`.
- "Проекты" — берет 3 якорных проекта и чек-листы из `projects.json`.
- "Карьера" — берет карьерные действия и трекер откликов из `career.json`.

100 основных задач считаются обязательным чек-листом. `supplementalTasks` — дополнительная практика; ее можно показывать отдельно и не включать в обязательный прогресс дня без отдельного продуктового решения.

## Проверка данных

После изменения файлов в `data/` нужно запускать:

```bash
node analyst-trainer/learning-plan/validate-data.mjs
```

Проверка подтверждает, что JSON читаются, MVP-разделы покрыты, обязательные задачи распределены по 7 месяцам, ссылки на модули известны, проекты имеют дефолтные UI-поля, а карьерный трек стартует с активных откликов в месяц 4.
