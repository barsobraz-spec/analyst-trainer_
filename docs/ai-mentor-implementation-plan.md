# AI-ментор: план реализации MVP

Источник: `docs/prd-ai-mentor-reference-check.md` и продуктовые решения от 2026-06-23.

## Принятые UX-решения

- Пустой или слишком слабый ответ не отправляется на проверку по эталону. Вместо этого доступна подсказка.
- Режим `hint` не получает полный `reference` и не получает эталонный SQL.
- SQL работает гибридно: подсказка доступна до автопроверки, объяснение ошибки доступно после неверной автопроверки.
- Перед отправкой показывается короткий preview и раскрываемые детали состава данных.
- Consent показывается один раз перед первым AI-запросом, затем хранится локально.
- Если AI-proxy не настроен, AI-блок виден, но disabled, с переходом в `#/settings`.
- AI-ответ мягко нормализуется, но базовый контракт обязателен.
- После feedback в MVP есть только действия "Исправить ответ" и "Повторить позже".
- Отправляются только поля, нужные конкретному режиму.
- Полный AI-feedback, полный ответ пользователя и полный prompt/context не сохраняются в MVP.
- Topic graph ограничивает реальные следующие шаги. UI строит ссылки только из локальных данных.
- Тон AI-ментора: спокойный наставник.
- `score` вторичен и показывается только в режимах полноценной проверки, например `reference_check`.

## Режимы MVP

| Режим | Когда доступен | Что отправляется | Что не отправляется |
| --- | --- | --- | --- |
| `hint` | Всегда, включая пустой ответ | условие, вопрос, тема, rubric, commonMistakes, черновик ответа если есть | полный reference, referenceSql |
| `reference_check` | Только при осмысленном ответе | условие, вопрос, ответ пользователя, reference, rubric, связанные темы | лишние поля кейса, полная история |
| `explain_error` | После проверки/сравнения, если есть ошибка | ответ пользователя, reference/rubric, контекст ошибки | прогресс, история |
| `sql_review` | После неверной SQL-автопроверки | userSql, referenceSql, schema, subtask prompt, autograderMessage | full feedback history |
| `next_step` | Для рекомендаций | агрегированный прогресс, topic refs, weak spots | полные тексты ответов |
| `business_review` | Для бизнес-выводов | ответ, rubric, reference/business criteria | лишние персональные данные |

## Последовательный план работ

1. Создать `core/mentorContext.js`.
   - Нормализовать режимы и входные данные кейса.
   - Собирать режимно-нужный контекст для 5.1 и 5.5.
   - Исключать reference из `hint`.
   - Блокировать `reference_check` для пустого/слишком слабого ответа.
   - Генерировать preview: короткая сводка и детали без сырого JSON.

2. Доработать `core/aiReview.js`.
   - Перейти от свободных `context/answer` к `mode/context/studentAnswer`.
   - Добавить consent helpers.
   - Добавить строгую, но терпимую нормализацию AI-ответа.
   - Обработать `400`, `401`, `429`, `502`, сетевые ошибки и невалидный JSON.

3. Создать `core/components/AiMentor.js`.
   - Disabled state без proxy.
   - Preview + details.
   - Consent перед первым запросом.
   - Loading/error/empty states.
   - Действия "Исправить ответ" и "Повторить позже".
   - Score вторичный, не показывать в `hint`, `explain_error`, `sql_review`, `next_step`.

4. Интегрировать в 5.1 Data Detective.
   - Режимы: `hint`, `reference_check`, `explain_error`, `next_step`.
   - Пустой ответ ведет к подсказке.
   - AI-feedback не раскрывает эталон и не финализирует прогресс.

5. Интегрировать в 5.5 SQL Investigation.
   - До автопроверки: только `hint`.
   - После mismatch/error: `sql_review` с последним `userSql`, текущей подзадачей, schema и autograderMessage.
   - SQL-автопроверка остается единственным источником зачета.

6. Добавить проверки.
   - Smoke для `mentorContext.js`.
   - Smoke для нормализации AI response.
   - Проверить no-proxy сценарий.
   - Запустить `learning-plan/validate-data.mjs` и `tests/smoke.py`.

## MVP не делает

- Не хранит полную историю AI-проверок.
- Не сохраняет полный текст ответа пользователя или AI-feedback.
- Не дает AI автоматически менять прогресс.
- Не проверяет README/портфолио.
- Не заменяет SQL-автопроверку.
